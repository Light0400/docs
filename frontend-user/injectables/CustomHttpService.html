Private getHeader
getHeader(customHeaders: any, isApi: boolean)

產生標準化的 HTTP headers，並根據是否為 API 請求動態補充必要的資訊。

  • 預設會加入：
    • X-TeamUp-Lang: 當前語言代碼。
    • X-HCS-Timezone: 使用者所在時區的 offset（分鐘）。
  • 若為 API 請求：
    • 預設加上 Content-Type: application/json（除非自訂 headers 已提供）。
    • 加上 Cache-Control: no-cache 與 Pragma: no-cache。
  • 若使用者已登入，則加上 Authorization: Bearer <token>。
Parameters :

customHeaders any No {}
  • 呼叫端傳入的自訂 headers。
isApi boolean No
  • 是否為 API 請求。
Returns : any

合併後的 headers 物件。

Private handleError
handleError(error: any)

處理 HTTP 錯誤，並根據狀態碼觸發相應的事件。

  • 若狀態碼為 401 或 403，會通知 authReqSubject 觸發重新登入流程。
  • 若狀態碼為 404，會通知 pageNotFoundSubject。
  • 其他錯誤則直接傳遞出去。
Parameters :
Name Type Optional Description
error any No
  • 錯誤物件，通常為 HttpErrorResponse。
Returns : Observable<never>

一個會立即拋出錯誤的 Observable。
Private invokeCapacitorHttp
invokeCapacitorHttp(method: "GET" | "POST" | "PUT" | "DELETE", options: HttpOptions)
Type parameters :
  • T

透過 Capacitor Http 插件發送實際的 HTTP 請求。

若回應的 HTTP 狀態碼不在 2xx 範圍內，會拋出 HttpErrorResponse，其中包含：

  • status: 狀態碼

  • statusText: 狀態文字

  • headers: 回應標頭

  • error: 錯誤內容（伺服器回應的資料）

  • url: 請求 URL

  • 根據輸入的 method 對應到 CapacitorHttp 的相應方法（get、post、put、delete）。

  • 透過 RxJS from 將 Promise 轉換成 Observable。

  • 使用 map 處理回應：

    • 狀態碼在 200~299 之間則回傳 res.data。
    • 其他狀態碼則建立並拋出 HttpErrorResponse。
Parameters :

  • HTTP 方法。
options HttpOptions No
  • 請求參數，包含 URL、headers 與資料。
Returns : Observable<T>

一個可觀察物件，發出成功回應的資料。
Private normalizeMomentInObject
normalizeMomentInObject(obj: any)

遞迴正規化物件中的日期或 moment 物件，將其轉換為 ISO 8601 字串。

  • 若輸入為 moment，呼叫 toISOString()。
  • 若輸入為 Date，呼叫 toISOString()。
  • 若輸入為陣列，則遞迴處理每個元素。
  • 若輸入為物件，則遞迴處理所有屬性。
  • 其他基本型別（string, number, boolean, null, undefined）則原樣返回。
Parameters :
Name Type Optional Description
obj any No
  • 欲正規化的資料，可以是物件、陣列、日期、moment 或基本型別。
Returns : any

正規化後的資料結構，日期與 moment 將轉為字串，其餘維持原狀。

Private send
send(method: "GET" | "POST" | "PUT" | "DELETE", options: HttpOptions)
Type parameters :
  • T

發送 HTTP 請求，並處理 URL、Header、Loading Mask 與錯誤攔截。

  • 會先檢查 URL 是否為 API 路徑，若是則透過 pathParser.apiParse 轉換。
  • 自動合併並補齊必要的 Header。
  • 在送出 API 請求時，若沒有 no-mask: yes header，會顯示 Loading Mask，直到請求完成。
  • 使用 RxJS 的 catchError 捕捉錯誤並交由 handleError 處理。
  • 使用 RxJS 的 finalize 確保請求完成後移除 Loading Mask。
Parameters :
Name Type Optional Description
method "GET" | "POST" | "PUT" | "DELETE" No
  • HTTP 方法。
options HttpOptions No
  • 請求選項。
Returns : Observable<T>

回傳一個可觀察物件，訂閱後會執行 HTTP 請求並發出結果。
